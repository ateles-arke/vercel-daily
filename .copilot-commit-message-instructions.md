Use Conventional Commits for every generated commit message.

Required format:
`<type>: <short imperative summary>`

Rules:

- Keep the type lowercase.
- Keep the summary concise and specific to the main change.
- Prefer a single clear change per commit message.
- Do not end the summary with a period.
- Use one of these types when they fit: feat, fix, refactor, perf, docs, chore.

Examples:

- feat: add responsive navbar
- fix: correct route params handling
- refactor: split news card component
- perf: optimize image loading
- docs: update README
- chore: upgrade dependencies
